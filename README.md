# TSU.MyMap — AlgoKotlin

Интерактивная карта-«песочница» кампуса Томского государственного университета. Шесть классических алгоритмов искусственного интеллекта и оптимизации, упакованных в нативное Android-приложение на Kotlin и Jetpack Compose.

Приложение задумано не как готовый продукт-навигатор, а как **визуальный демонстратор алгоритмов**: каждая фича на главном экране — это отдельный, самостоятельно реализованный алгоритм, работающий поверх одной и той же карты кампуса (матрица проходимости 154×100).

## ✨ Что внутри

| # | Раздел | Алгоритм | Задача |
|---|--------|----------|--------|
| 1 | **Карта кампуса** | A* | Поиск пути «отсюда сюда» по объектам кампуса |
| 2 | **Навигация** | A* (с пошаговой визуализацией) | Демонстрация порядка раскрытия вершин |
| 3 | **Где поесть?** | Генетический алгоритм | Маршрут, покрывающий желаемые типы еды |
| 4 | **Кластеризация** | K-Means | Группировка точек интереса по зонам |
| 5 | **Коворкинги** | Муравьиный алгоритм (ACO) | Решение задачи коммивояжёра по учебным зонам |
| 6 | **Куда пойти на обед?** | Дерево решений (ID3) | Рекомендация заведения по 6 признакам |
| 7 | **Распознавание цифр** | Многослойный перцептрон | Inference нейросети 2500→256→64→10 на устройстве |

## 🛠 Технологический стек

- **Язык:** Kotlin `2.2.10`
- **UI:** Jetpack Compose + Material 3
- **Архитектура:** Single-Activity, навигация через состояние и `AnimatedContent`
- **Async:** Kotlin Coroutines (`Dispatchers.Default` для тяжёлых вычислений)
- **Min/Target SDK:** 24 / 36
- **Доп. зависимости:** `androidx.core:core-splashscreen`, TensorFlow Lite (как зависимость; инференс выполняется собственной реализацией на чистом Kotlin)
- **Сборка:** Gradle Kotlin DSL, Version Catalog (`libs.versions.toml`)

Приложение работает **полностью офлайн** — никаких сетевых запросов и внешних API. Все данные (карта, веса нейросети, обучающая выборка дерева решений) лежат в `assets/`.

## 🧠 Реализованные алгоритмы

### 1. A\* — поиск кратчайшего пути

> Классический A\* на 4-связной сетке с манхэттенской эвристикой `h = |Δx| + |Δy|`.

- Открытый и закрытый списки, восстановление пути по родительским ссылкам.
- Вспомогательная функция **`findNearestWalkable`** — расширяющийся «ромб» по `dist = |Δx| + |Δy|`, который притягивает выбранную пользователем точку (например, здание на карте) к ближайшей проходимой клетке.
- Отдельная вариация **`astarWithSteps`** возвращает не только путь, но и порядок раскрытия вершин — это позволяет на экране Навигации проигрывать построение маршрута как анимацию.

### 2. K-Means — кластеризация точек кампуса

- Случайная инициализация центроидов из самих точек.
- Поддерживаются **три метрики расстояния**, переключаемые в UI: евклидова, манхэттенская, Чебышёва.
- Ранняя остановка по сходимости (если назначения не изменились между итерациями).
- Пользователь может тапом по карте добавлять собственные точки и подбирать `k`.

### 3. Генетический алгоритм — маршрут за едой

Задача нетривиальна: нужно не просто кратчайший маршрут, а покрытие **множества желаемых типов еды** (`coffee`, `pancakes`, `full_meal`, `snack`) при минимальной длине пути. По сути — гибрид TSP и Set-Cover.

- **Особь** — упорядоченный список индексов заведений.
- **Fitness** — суммарное BFS-расстояние по сетке кампуса + бесконечный штраф за непокрытие хотя бы одного из выбранных типов еды.
- **Селекция** — турнирная (k=4).
- **Кроссовер** — собирает потомка из объединённого пула генов родителей, жадно добавляя точки, которые покрывают ещё не закрытые типы еды.
- **Мутация** — своп двух генов или замена точки на «эквивалентную» по меню, с последующим прунингом избыточных стоп-точек.
- **Элитизм** — лучшая особь поколения переходит в следующее без изменений.
- Расстояния между всеми парами точек предвычисляются один раз через **BFS-карты** (`bfsDistanceMap`) — это типичная оптимизация: вместо `O(n²)` повторных поисков пути алгоритм работает на кэшированной матрице.

### 4. Муравьиный алгоритм (ACO TSP) — обход коворкингов

Реализация Ant System в каноническом виде:

- **Феромонная матрица** инициализируется единицами; в начале каждой итерации испаряется на коэффициент `(1 − ρ)`.
- Вероятность перехода муравья из `i` в `j` пропорциональна `τᵢⱼ^α · ηᵢⱼ^β`, где `η = comfort / distance` — эвристика «привлекательности» точки.
- После прохода всех муравьёв феромон откладывается обратно: `Δτ = q / L_tour`.
- Параметры по умолчанию: `α = 1.0`, `β = 3.0`, `evaporation = 0.5`, 80 итераций по 20 муравьёв.
- На каждой итерации сохраняется лучшее решение — UI отрисовывает кривую сходимости.

### 5. Дерево решений (ID3) — куда пойти на обед

- Чтение обучающей выборки из CSV (`food_decisions.csv`, 31 пример).
- Расчёт **энтропии Шеннона** и **information gain** для выбора атрибута на каждом уровне.
- Рекурсивное построение через `sealed class TreeNode` (`Leaf | Branch`) — алгебраическая структура данных, идиоматичная для Kotlin.
- При классификации сохраняется **полный путь по дереву** (`DecisionPath`) — пользователь видит не только результат, но и какие именно ответы на какие признаки к нему привели. Объяснимая модель «из коробки».
- Если какое-то значение признака не встречалось в обучающей выборке, узел отдаёт **majority-class fallback**.
- Для отладки доступна функция `treeToText` — сериализация всего дерева в текст с отступами.

### 6. Нейросеть для распознавания цифр

> Полностью **самописный inference-движок** на чистом Kotlin — без TensorFlow Lite, без ND4J, без OpenCV.

- **Архитектура:** трёхслойный MLP `2500 → 256 → 64 → 10`, ReLU на скрытых слоях, argmax на выходе.
- **Веса** загружаются из бинарных файлов `w1.bin`, `b1.bin`, `w2.bin`, `b2.bin`, `w3.bin`, `b3.bin` (формат: little-endian `float32`, читается через `ByteBuffer`).
- **Forward pass** реализован как обычные циклы по матрицам — это нагляднее, чем дёргать абстрактный фреймворк.
- **Препроцессинг** входа выполняет центрирование рисунка по центроиду активных пикселей и масштабирование к целевому центру 50×50 — приём, существенно поднимающий качество распознавания «сдвинутых» цифр.
- Inference запускается через `withContext(Dispatchers.Default)`, чтобы не блокировать UI-поток.

## 📱 Приёмы мобильной разработки

Приложение — это ещё и небольшая витрина возможностей **Jetpack Compose**:

- **Декларативный UI** на Compose, без единого XML-лейаута для экранов.
- **Single-Activity навигация** через `mutableStateOf` и `AnimatedContent` с кастомными `transitionSpec` (slide + fade в обе стороны в зависимости от направления перехода).
- **Жесты на карте:**
  - `detectTransformGestures` — pinch-to-zoom и drag одной рукой;
  - `detectTapGestures` — выбор точек старта/финиша;
  - `awaitPointerEventScope` — непрерывное рисование пальцем на холсте нейросети с кистью 3×3.
- **Кастомная отрисовка** через `Canvas` и `drawIntoCanvas` — карта, маршруты, кластеры, дерево решений, пиксельная сетка для рисования.
- **Анимации:**
  - переходы между экранами (`AnimatedContent`);
  - всплывающие нижние карточки (`AnimatedVisibility` + `slideInVertically/fadeIn`);
  - **пошаговая анимация работы A\***: клетки, посещённые алгоритмом, постепенно «загораются» на карте, после чего отрисовывается итоговый путь.
- **Modal Bottom Sheet** для информационных карточек объектов кампуса.
- **Адаптивная вёрстка** — RouteScreen реагирует на `LocalConfiguration` и переключает раскладку в landscape.
- **Edge-to-edge** + **SplashScreen API**.
- **Полная локализация** через `strings.xml` со всеми форматными аргументами — никакого хардкода в Composable.
- **Material 3 + кастомная тема** в брендовых синих цветах ТГУ.
- **Загрузка ресурсов** напрямую из `assets/` (`context.assets.open(...)`) с кэшированием в `remember`, чтобы тяжёлые файлы (карта, веса нейросети) не перечитывались при перерисовке.

## 📂 Структура проекта

```
app/src/main/
├── java/com/example/algokotlinapp/
│   ├── MainActivity.kt           — Single-Activity host + анимации переходов
│   ├── MainMenuScreen.kt         — главное меню + bottom sheet «О приложении»
│   ├── CampusMapScreen.kt        — интерактивная карта + словарь POI
│   ├── RouteScreen.kt            — A* с визуализацией шагов
│   ├── FoodScreen.kt             — генетический алгоритм
│   ├── KMeansScreen.kt           — кластеризация
│   ├── CoworkingScreen.kt        — муравьиный алгоритм
│   ├── DecisionTreeScreen.kt     — дерево решений
│   ├── NeuralNetScreen.kt        — рисование + inference нейросети
│   ├── algorithms/               — чистая логика, без зависимостей от Compose
│   │   ├── AStarAlgorithm.kt
│   │   ├── KMeansAlgorithm.kt
│   │   ├── GeneticAlgorithm.kt
│   │   ├── AntAlgorithm.kt
│   │   └── DecisionTree.kt
│   ├── model/Node.kt             — узел A*
│   └── ui/theme/                 — Material 3 тема ТГУ
│
├── assets/
│   ├── tsu_campus_matrix.txt     — карта кампуса (матрица 0/1)
│   ├── TSU_map_naming.txt        — наименования объектов
│   ├── food_decisions.csv        — обучающая выборка для ID3
│   ├── w1.bin, b1.bin            — веса слоя 1 нейросети
│   ├── w2.bin, b2.bin            — веса слоя 2
│   └── (w3.bin, b3.bin)          — веса слоя 3
│
└── res/                          — ресурсы, drawable, strings, theme
```

Логика алгоритмов **намеренно отделена** от UI: пакет `algorithms/` не зависит от Compose и Android SDK — его можно переиспользовать в десктопном или серверном Kotlin-проекте.

## 📊 Источники данных

- **Карта кампуса** — собственная разметка матрицы проходимости на основе спутниковой карты ТГУ (Томск).
- **Словарь объектов** (`CampusDictionary`) — около 35 точек интереса с координатами и категориями (остановки, еда, магазины, учебные корпуса, коворкинги, памятники и др.).
- **Обучающая выборка ID3** — 31 размеченный пример «местоположение / бюджет / время / тип еды / толерантность к очереди / погода → рекомендация».
- **Веса нейросети** — обучены отдельно (вне приложения) на датасете рукописных цифр и сериализованы в формате `float32 little-endian`.

## 🎯 Идея проекта

Это учебный проект курса HITS, в котором изучение классических алгоритмов было совмещено с практикой нативной Android-разработки. Каждый алгоритм здесь — не «чёрный ящик из библиотеки», а написанный с нуля код, который можно открыть, прочитать и модифицировать. Карта кампуса служит общей предметной областью, на которой все алгоритмы получают наглядную интерпретацию: A\* строит дорожку до корпуса, генетика собирает обед, муравьи обходят коворкинги, нейросеть распознаёт цифры — и всё это в одном приложении.
