# AlgoKotlinApp — Архитектура проекта

Мобильное приложение для студентов ТГУ: навигация и рекомендации по кафе на кампусе.
Модуль 2 — Алгоритмы прикладных вычислений.

---

## Структура файлов

```
com.example.algokotlinapp/
│
├── algorithms/                  # чистая логика — никакого Android
│   ├── AStarAlgorithm.kt        # поиск кратчайшего пути на сетке
│   ├── KMeansAlgorithm.kt       # кластеризация точек на карте
│   ├── GeneticAlgorithm.kt      # оптимальный маршрут покупки еды
│   ├── AntColonyAlgorithm.kt    # маршрут обхода кафе (коммивояжёр)
│   ├── DecisionTreeAlgorithm.kt # рекомендация кафе по параметрам
│   └── NeuralNetwork.kt         # распознавание нарисованной цифры
│
├── model/                       # классы данных
│   ├── Node.kt                  # клетка карты (x, y, стоимость пути)
│   └── Cafe.kt                  # кафе: название + координата на сетке
│
├── data/
│   └── CampusData.kt            # карта кампуса + список кафе (данные)
│
└── ui/
    ├── MainActivity.kt          # главное меню — 6 кнопок
    ├── AStarActivity.kt
    ├── KMeansActivity.kt
    ├── GeneticActivity.kt
    ├── AntColonyActivity.kt
    ├── DecisionTreeActivity.kt
    └── NeuralNetActivity.kt
```

---

## Три слоя

### 1. `algorithms/` + `model/` — логика

- Чистый Kotlin, без единого импорта из `android.*`
- Принимает данные на вход, возвращает результат
- Можно запустить и проверить в консоли без эмулятора
- Можно покрыть тестами независимо от UI

### 2. `data/` — данные

- Карта кампуса в виде двумерной сетки (0 = проход, 1 = стена/здание)
- Список кафе с названиями и координатами на сетке
- Обучающие данные для дерева решений
- На старте всё хардкодом; позже можно заменить на CSV / JSON

### 3. `ui/` — интерфейс

- Каждый алгоритм — отдельный экран (Activity)
- UI только берёт результат алгоритма и рисует его
- Никакой бизнес-логики внутри Activity

---

## Поток данных

```
CampusData  →  Algorithm  →  Result  →  Activity рисует
(карта, кафе)  (считает)   (путь/кластеры)  (на экране)
```

Этот поток одинаков для всех шести алгоритмов.

---

## Навигация между экранами

```
MainActivity (меню)
    ├── → AStarActivity         A* — маршрут до кафе
    ├── → KMeansActivity        K-means — зоны питания
    ├── → GeneticActivity       Генетический — маршрут покупки
    ├── → AntColonyActivity     Муравьиный — обход кампуса
    ├── → DecisionTreeActivity  Дерево решений — выбор кафе
    └── → NeuralNetActivity     Нейросеть — распознавание цифры
```

## Алгоритмы

| Алгоритм | Файл | Задача |
|---|---|---|
| A* | `AStarAlgorithm.kt` | Кратчайший путь от пользователя до кафе на сетке кампуса |
| K-means | `KMeansAlgorithm.kt` | Группировка кафе по зонам, определение кластеров питания |
| Генетический | `GeneticAlgorithm.kt` | Оптимальный маршрут покупки еды с ограничениями |
| Муравьиный | `AntColonyAlgorithm.kt` | Обход всех нужных кафе с минимальным путём |
| Дерево решений | `DecisionTreeAlgorithm.kt` | Рекомендация кафе по: бюджету, времени, типу еды, очереди |
| Нейросеть | `NeuralNetwork.kt` | Распознавание цифры, нарисованной пальцем на экране |

## Главное правило

**Алгоритм не знает про Android.**

Файлы в `algorithms/` содержат только математику и логику. Они не знают ничего про экраны, кнопки, Android SDK. Это позволяет:

- Написать и проверить алгоритм в консоли (прогресс-чек)
- Потом подключить к Android UI без переписывания логики
- В будущем переиспользовать код на другой платформе

## Порядок разработки

1. **Прогресс-чек** — `Node.kt` + `AStarAlgorithm.kt`, вызов из `MainActivity`, вывод в Logcat
2. Добавить `Cafe.kt` + `CampusData.kt` — реальные данные кампуса
3. Сделать `AStarActivity` с отрисовкой карты на экране
4. По той же схеме реализовать остальные алгоритмы
5. Сделать `MainActivity` с меню навигации
